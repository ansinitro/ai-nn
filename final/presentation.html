📊 Vector Database — FAISS

Index Configuration

  • Type: IndexFlatIP
  • Metric: Inner Product (= cosine for normalized vectors)
  • Vectors: 1700
  • Dimensions: 768
  • Size: 5.0 MB

Why FAISS Flat?

  • Exact search (no approximation loss)
  • Fast for <10K vectors
  • CPU-based (saves GPU for LLM)
  • Sub-millisecond query time
Sanity Check: Query chunk[0] → returns itself with score 1.000 ✅

📏 Evaluation Methodology

Metrics

  • ROUGE-1 — unigram overlap
  • ROUGE-L — longest common subsequence
  • BERTScore F1 — semantic similarity using RoBERTa

Test Set

  • 15 question-answer pairs
  • 10 diverse ML topics
  • Manually crafted reference answers
  • Stored in eval/test_questions.json

Evaluation Process

Each question → RAG pipeline → predicted answer → compare with reference using ROUGE + BERTScore → log to CSV with sources and timing data
