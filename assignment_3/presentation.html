Подготовка последовательностей

Данные Jena Weather преобразованы в формат supervised learning с использованием скользящего окна.

Формирование окон

  • Для каждого временного шага t создаётся окно из 24 предыдущих наблюдений (4 часа при интервале 10 мин).
  • Целевая переменная — T(degC) на шаг t.
  • Итоговый массив X имеет размерность (samples, 24, features).

Разделение данных

  • Train: 70% (первые ~294 тыс. записей)
  • Validation: 15% (~63 тыс. записей)
  • Test: 15% (~63 тыс. записей)
  • Хронологический порядок сохранён
Стандартизация: StandardScaler обучен только на тренировочной выборке, затем применён ко всем split'ам.

Схема формирования окна

t-24 → [x₁, x₂, ..., xₙ]
t-23 → [x₁, x₂, ..., xₙ]
... ⋮
t-1 → [x₁, x₂, ..., xₙ]
t → ŷ = T(degC) ← prediction target

Каждый sample = матрица 24 × N_features

Почему LSTM? В отличие от MLP, LSTM обрабатывает данные последовательно, запоминая долгосрочные зависимости через механизм ячеек памяти (forget, input, output gates).

LSTM Cell Architecture

Forget Gate:
$f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$
Input Gate:
$i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)$
$\tilde{C}_t = \tanh(W_C \cdot [h_{t-1}, x_t] + b_C)$
Cell Update:
$C_t = f_t \ast C_{t-1} + i_t \ast \tilde{C}_t$
Output Gate:
$o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)$
$h_t = o_t \ast \tanh(C_t)$

Сравнение: MLP vs DNN vs LSTM

Количественное сравнение трёх архитектур на тестовой выборке.

Модель Архитектура Test RMSE Test MAE
MLP 3 скрытых слоя (64→32→16) 0.650 °C 0.450 °C
DNN 5 скрытых слоёв (128→64→32→16→8) 0.620 °C 0.430 °C
LSTM 2 слоя LSTM (64→32) + Dense(1) 0.186 °C 0.121 °C
Ключевое преимущество LSTM: Модель явно учитывает порядок наблюдений и долгосрочные зависимости, в то время как MLP/DNN обрабатывают каждое наблюдение независимо.
