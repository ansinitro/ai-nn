The Theoretical Task & Challenge

In Assignment 2, we predicted Temperature using Multi-Layer Perceptrons (MLP). However, MLPs struggle with temporal data because they lack memory—each row is treated as an isolated independent event.

LSTM Paradigm Shift

  • Recurrent State: RNNs natively pass their hidden state $h_t$ continuously across time, remembering what happened hours ago.
  • Gate Mechanisms: Long Short-Term Memory cells specifically utilize Forget, Input, and Output gates to filter out noise while preserving distinct seasonal trends.

Architecture & Computational Strategy

Optimal Hyperparameter Choice

We avoided blindly stacking 256 units. Model complexity must align tightly with signal clarity.

  • LSTM(64): Weather signals display diurnal (daily) cyclical rhythm. 64 units natively acts as a temporal compression chamber, powerful enough to recognize heat spikes but constrained enough not to overfit standard statistical noise.
  • Dense(32) Adapter: Translates the raw latent dimension generated by our LSTM cell into an explicit, deterministic physical value.
  • Linear Activation: Directly mapped to degrees Celsius output.
# The Defined Keras Core Architecture
model = Sequential([
   Input(shape=(timesteps, features)),
   LSTM(64, return_sequences=False),
   Dense(32, activation='relu'),
   Dense(1, activation='linear')
])

model.compile(optimizer='adam', loss='mse')

Note: We implemented the crucial EarlyStopping(patience=3) callback monitoring val_loss to cleanly shut down training preventing memory-memorization over 15 Epochs.

Hyperparameter Test: The Length of History

How far back should an AI look? Comparing 12, 24, and 48 timestep "Memory Windows".

Empirical Results

  • Window = 12 (2 hours): Near-term precision is good but occasionally lacks the full context of whether it's dawn or dusk.
  • Window = 24 (4 hours): The Sweet Spot. Provides the exact context the network needs to calculate heating/cooling inertia without dragging obsolete noise.
  • Window = 48 (8 hours): Slight performance degradation and immensely slower memory processing! Storing data across an entire shift injects irrelevant old noise that the "Forget" gate has to work harder to ignore.

Final Conclusion: LSTM vs MLP Baseline

Why the LSTM is Superior

1. Eradication of Manual "Lags":

In Stage 2 (MLP), we were forced to inject arbitrary artificial `lag_1` and `lag_2` columns. LSTMs generate sequential embeddings strictly naturally via 3D `(samples, time, features)` representations.

2. Sequential Prowess:

The LSTM's RMSE dropped aggressively closer to absolute zero mapping limits compared to standard MLPs. Time is inherently structured, and LSTMs mathematically observe Time.
