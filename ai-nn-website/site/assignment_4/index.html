Task Framing

  • The assignment text specifies Appliances, but the provided processed dataset is the same weather dataset used in assignments 2 and 3.
  • To keep the comparison valid, the binary label was created from T (degC): class 1 means the target value is above the global median 9.42.
  • The resulting class balance is almost perfect: positive share is 49.97%, so no class weights were needed.
  • Sequence construction followed the previous time-series setup: 24 past steps predict the next-step class.

This preserves consistency with assignment 2 (MLP regression) and assignment 3 (LSTM regression), while still satisfying the CNN classification requirement.

The median split produces a nearly balanced dataset, which makes accuracy, precision, recall, and F1 directly interpretable.

Architecture & Search

  • Model pipeline: Conv1D - MaxPooling1D - Flatten - Dense(64) - Dense(1, sigmoid).
  • Loss: binary_crossentropy. Optimizer: Adam.
  • Training metrics: accuracy, precision, recall.
  • Early stopping monitored val_loss with patience 3.

All four CNN configurations performed strongly, but 64 filters / kernel 3 produced the best validation F1 and was saved as cnn_classifier.h5.

Training is stable, converges quickly, and does not show visible overfitting on the validation split.

The confusion matrix confirms that both classes are learned well, with very few false positives and false negatives.

Predicted probabilities track regime switches sharply, and most decisions are far from the 0.5 threshold.

Main Conclusions

  • The assignment requirements were completed: binary label creation, 1D CNN, chronological splits, confusion matrix, and metric-based evaluation.
  • The CNN clearly improves over the classical baseline and reaches F1 = 0.9933 on the test set.
  • Against previous assignments, the CNN is competitive with the LSTM in binary mode, but it does not surpass the saved MLP threshold transfer.
  • For defense, the most important explanation is that feature-engineered lags from assignment 1 still strongly favor feed-forward models on this specific dataset.
