Relevance & Problem

The Stefan problem describes phase-change phenomena — ice melting, metal solidification, crystal growth — where the boundary between two phases moves over time.

  • PINNs (Physics-Informed Neural Networks) solve these as a mesh-free alternative to classical methods
  • Problem: Real sensor data is always contaminated by noise, degrading PINN accuracy
  • Gap: No prior work systematically evaluated preprocessing the data to reduce noise before PINN training

Hypothesis

Applying noise filtering techniques (Gaussian smoothing, Total Variation denoising, Wavelet thresholding, Non-Local Means, and ensemble methods) as a preprocessing step to noisy measurement data will significantly improve the accuracy of Physics-Informed Neural Network solutions for inverse Stefan problems, without requiring any modifications to the PINN architecture or training procedure.

Reasoning

  • Data loss term: $\mathcal{L}_{\text{data}} = \frac{1}{M}\sum_{i=1}^{M}(\hat{u}_i - \tilde{u}_i)^2$ — noisy $\tilde{u}_i$ forces the PINN to fit noise
  • Filtering cleans $\tilde{u}_i$ before training → cleaner targets → better solutions
  • Different filters will be optimal at different noise levels

Mathematical Model

Two-phase Stefan problem on $x \in [0,1]$, $t \in [0,1]$:

Phase 1:  $\frac{\partial u_1}{\partial t} = k_1 \frac{\partial^2 u_1}{\partial x^2}, \quad 0 < x < s(t)$
Phase 2:  $\frac{\partial u_2}{\partial t} = k_2 \frac{\partial^2 u_2}{\partial x^2}, \quad s(t) < x < 1$
Stefan condition:  $\dot{s}(t) = k_1 \frac{\partial u_1}{\partial x}\Big|_{x=s(t)} - k_2 \frac{\partial u_2}{\partial x}\Big|_{x=s(t)}$

Thermal diffusivities: $k_1 = 2.0$, $k_2 = 1.0$. Exact solution via similarity variables $\eta = x/\sqrt{t}$.

Noisy Data Model

$\tilde{u}_i = u_{\text{exact}}(x_i, t_i) + \sigma \cdot \epsilon_i, \quad \epsilon_i \sim \mathcal{N}(0,1)$

Noise Levels

σ = 1% Mild — high-precision lab sensors
σ = 5% Moderate — industrial thermocouples
σ = 10% Severe — harsh environments

PINN Architecture

Neural Networks

  • Temperature net $u_\theta(x,t) \to (u_1, u_2)$:
    $[2, 100, 100, 100, 2]$ with $\tanh$ activation
  • Boundary net $s_\beta(t) \to s$:
    $[1, 100, 100, 100, 1]$ with $\tanh$ activation

Composite Loss

$\mathcal{L} = \mathcal{L}_{\text{PDE}} + c_{\text{data}} \cdot \mathcal{L}_{\text{data}} + \mathcal{L}_{\text{Stefan}}$
  • M1: Fixed $c_{\text{data}} = 1.0$
  • M2: Adaptive $c_{\text{data}}$ via NTK gradient balancing

Noise Filtering Methods

Six methods implemented + NoFilter baseline. The core idea: clean the data before PINN training.

# Filter Type Key Parameters
1 Gaussian Smoothing Classical $\sigma_{\text{kern}} = \max(0.5, 2\sigma_{\text{noise}})$
2 Total Variation (TV) Edge-preserving Chambolle algorithm, $\lambda_{\text{TV}} = 0.1 \cdot \sigma_{\text{noise}}$
3 Wavelet Thresholding Multi-scale Daubechies-4 (db4), BayesShrink soft thresholding
4 Non-Local Means (NLM) Patch-based Patch size & search window adapted to $\sigma_{\text{noise}}$
5 PhysDnCNN Deep Learning Self-supervised DnCNN (Noise2Self approach)
6 Ensemble Hybrid $0.4 \cdot u_{\text{NLM}} + 0.3 \cdot u_{\text{Gauss}} + 0.3 \cdot u_{\text{Wavelet}}$

Improvement Analysis

Percentage improvement in L₂(u) error relative to the NoFilter baseline across all configurations.

  • 5 out of 6 filters reduce error vs. baseline
  • Classical methods consistently outperform the deep learning denoiser
  • Finer grids (M=100) benefit more from filtering than coarser grids (M=50)
  • Ensemble provides the most robust improvement across all regimes

Key Insight: Simple classical signal processing outperforms deep learning denoisers for small scientific datasets.

Practical Recommendations

Guidelines for practitioners working with noisy Stefan problem data:

Noise Level Recommended Filter Expected Improvement
Low (σ ≤ 1%) Gaussian Smoothing ~37% error reduction
Moderate (σ ≈ 5%) Ensemble (NLM + Gauss + Wavelet) ~52% error reduction
High (σ ≥ 10%) TV or Ensemble ~61% error reduction
Unknown Ensemble (default choice) Robust across all regimes

💡 For practitioners without time to tune filter parameters:
The Ensemble filter is recommended as the default choice — consistently strong across all noise levels without requiring prior noise knowledge.
