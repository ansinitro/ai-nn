Siamese Network Architecture

  • Two input branches share a single 1D-CNN embedding network (tied weights).
  • Each branch maps a window (24, 26) → embedding vector of size 128.
  • A Lambda layer computes the Euclidean distance between the two embeddings.
  • Output: scalar distance ∈ [0, ∞) used for anomaly scoring.
  • Backbone: Conv1D(64,k=3) → Pool → Conv1D(128,k=3) → Pool → Flatten → Dense(256) → Dense(128).

Pair Construction

Similar Pairs (y = 0)

  • Both windows: HighConsumption = 0 (normal).
  • Goal: push distance → 0.

Dissimilar Pairs (y = 1)

  • One normal window + one anomalous window.
  • Goal: push distance > margin (M = 1.0).

Training Set

  • 60,000 balanced pairs (30k similar + 30k dissimilar).
  • 15,000 validation pairs.
  • Sampled from chronological train/val splits.

Anomaly rate: ~50% (balanced binary target). Both classes well-represented in pairs.

Contrastive Loss

The loss function penalizes the network differently for similar vs. dissimilar pairs:

L = y · d² + (1−y) · max(M − d, 0)²

d = Euclidean distance  |  M = margin = 1.0  |  y ∈ {0,1}

  • y = 0 (similar): loss = d² → minimise distance between normal pairs.
  • y = 1 (dissimilar): loss = max(M−d, 0)² → push distance beyond margin.
  • Implemented from scratch using tf.keras custom loss function.
  • Margin M=1.0 creates a clear decision boundary in embedding space.

Inference: Anomaly Scoring

Step 1 – Build Reference Library

  • Extract embeddings for 5,000 sampled normal training windows.

Step 2 – Score Test Windows

  • For each test window: compute embedding, then find minimum Euclidean distance to any reference normal embedding (kNN, k=1).
  • High distance → window is anomalous.

Step 3 – Threshold Selection

  • Sweep thresholds on validation kNN distances.
  • Select threshold maximising F1 on validation labels.
  • Best threshold: 0.2571 (val F1 = 0.8538).

Contrastive loss decreases steadily over 20 epochs. Train and validation loss track closely with no overfitting — the shared weights generalise well.

Architecture Recommendations

🏆 Forecasting (Regression)

  • Best: MLP (A2) — RMSE 0.019, MAE 0.013. Compact but powerful.
  • LSTM (A3) captures temporal dependencies but underperforms MLP on this dataset.

🏆 Classification

  • Best: MLP (A2) when used for binary classification (F1 = 0.999).
  • CNN (A4) is best sequence classifier at F1 = 0.9933.

🏆 Anomaly Detection

  • Best: Siamese (A6) — AUC 0.90, no anomaly labels needed at training.
  • Autoencoder reconstruction error (A5) is a complementary approach.

Conclusions

  • A Siamese 1D-CNN with contrastive loss successfully learns a metric space where anomalous windows are far from the normal reference set.
  • AUC of 0.8989 and F1 of 0.8136 demonstrate strong anomaly detection without explicit anomaly labels during training.
  • The kNN scoring strategy (distance to nearest normal embedding) provides a simple, interpretable anomaly score.
  • Supervised classifiers (MLP, CNN) remain superior for binary classification when labels are available.
  • The Autoencoder (A5) and Siamese (A6) complement each other: AE flags global pattern deviations; Siamese flags contextual distance from normal.
