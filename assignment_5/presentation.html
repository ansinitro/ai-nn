Autoencoder Architecture

  • Input: flattened sliding windows — 24 timesteps × 26 features = 624 dimensions.
  • Symmetric fully-connected encoder–decoder with a 64-unit bottleneck.
  • Compression ratio: 624 → 64 (≈ 10×).
  • Loss: MSE. Optimizer: Adam.
  • Trained on clean data only (input == target), so the model learns the manifold of normal sensor readings.
  • Early stopping on val_loss, patience = 5.
Input (624) → Dense 256 → Dense 128 → Bottleneck 64 → Dense 128 → Dense 256 → Output (624)

Total trainable parameters: 402,864 (1.54 MB)

Experiment Design

Step 1 — Train on Clean

  • Autoencoder learns to reconstruct clean windows faithfully.

Step 2 — Inject Noise

  • Gaussian noise N(0, σ) added to each feature in every test window.
  • Primary experiment: σ = 0.1 (on standardised data).

Step 3 — Denoise

  • Noisy windows passed through the autoencoder → reconstructed output.

Step 4 — Classify

  • CNN classifier from Assignment 4 applied to three variants: clean, noisy, and denoised.

Autoencoder converges within ~15 epochs. Train and validation loss track closely — no overfitting.

Conclusions

  • The autoencoder was successfully trained on clean data and learned to reconstruct sensor windows with low MSE (0.0139).
  • Signal-level visualisation confirms faithful denoising — the reconstructed signal closely tracks the original while filtering noise.
  • At low noise (σ ≤ 0.1), the CNN is already robust and the autoencoder provides no net benefit.
  • At moderate-to-high noise (σ ≥ 0.2), the autoencoder consistently improves classification accuracy and F1 score.
  • The benefit grows with noise intensity: at σ = 0.5, the autoencoder recovers +2.6% accuracy on average.
  • This proves the autoencoder's ability to improve model robustness against sensor failures and data corruption.
